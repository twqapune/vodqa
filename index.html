About

vodQA - Value Oriented Discussions for Quality Analysts, is a forum that sports the motto - "Come learn something new". The forum offers a unique platform for the software testing industry to strengthen the QA community through knowledge sharing.

So far, we have successfully hosted more than 20 editions of vodQA across all Thoughtworks India offices. vodQA aims at taking you to another level altogether, where you get a wider perspective of testing. Let us meet more evangelists, to talk and learn about the vast spectrum of testing realms.

Welcome Note

OAuth 2.0
by Shubhashree Deshpande and Hari Om Gautam

OAuth is buzzing all around in recent times..But actually people are confused on what actually 'OAuth' is?. - Some people think it as secure way of Authentication..But Actually 'Oauth' is a Standard Authentication Protocol/ Delegation Protocol which allows user to login to third party application without actually sharing 'UserName' and 'Password'. - It works on 'Token'

04.05 - 04:45 PM

CI/CD using Builkite
by Kritika Vohra and Pooja Goyal

The culture of TW enables us to act as a gel and easily upskill to fit between client requirements and deliverables. This helps us adapt to different tools and tech stacks in our projects. We as QA come across a lot of CI/CD tools in our projects. One such tool is Buildkite, our talk will give a glimpse of Buildkite tool that is not used to so widely but is a super powerful tool for creating continuous integration pipelines for projects.

4:55 - 5:40 PM

Integrate Postman Test to CI
by Sachin Gupta

API testing is one of the crucial parts of product development, API component is often developed before the UI component which allows for quick feedback, API testing is also a lot cheaper than manual E2E testing, If we start automating APIs in the early stage, it will definitely help to test the APIs quickly, also it helps to verify that any code changes wonâ€™t introduce regressions in Production, to develop the automation framework for API testing will take some time, then how to automate APIs quickly? We can automate APIs with the help of postman collections and can run those collections in CI on daily basis, We often use the Postman tool for validating the response of APIs. This talk will walk through, how to automate APIs quickly and how to execute API test cases on daily basis in CI pipeline (Jenkins and GitHub Actions)
